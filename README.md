# Terraform Seguro 2025: DevSecOps & Azure sem Vulnerabilidades

Bem-vindo ao repositório oficial do curso!

Este repositório contém todos os códigos, laboratórios e exemplos práticos utilizados durante as aulas. Aqui você encontrará projetos de Infraestrutura como Código (IaC) focados em segurança, automação e boas práticas de mercado.

Este curso foi desenhado para levar você do zero ao avançado em Terraform, com foco especial em segurança (DevSecOps) e ambientes Azure e AWS.

## Estrutura do Repositório

O conteúdo está organizado de forma modular para facilitar seus estudos:

### 01-terraform-aws
Laboratórios práticos focados no provisionamento de recursos na **Amazon Web Services (AWS)**.
- Criação de Buckets S3 seguros.
- Configuração de EC2 e redes.
- Boas práticas de segurança na AWS.

### 02-terraform-azure
Laboratórios práticos focados no provisionamento de recursos no **Microsoft Azure**.
- Gerenciamento de Resource Groups.
- Provisionamento de recursos com segurança.
- Integração com ferramentas de DevSecOps.

### 03-boto3-aws
Exemplos de automação e scripts utilizando **Python** e a biblioteca **Boto3** para interagir com a AWS programaticamente.

### info
Materiais de apoio, explicações didáticas e resumos dos conceitos abordados nos laboratórios.

## Como Utilizar este Repositório

1. **Clone o repositório:**
   ```bash
   git clone https://github.com/iesodias/iac-pro.git
   cd iac-pro
   ```

2. **Escolha um laboratório:**
   Navegue até a pasta do módulo que deseja estudar (ex: `cd 01-terraform-aws/lab-01-s3`).

3. **Siga as instruções:**
   Cada laboratório possui seu próprio `README.md` com o passo a passo detalhado para execução.

## Pré-requisitos

Para executar os laboratórios, você precisará das seguintes ferramentas instaladas:

- [Terraform](https://www.terraform.io/downloads) (v1.0+)
- [AWS CLI](https://aws.amazon.com/cli/) (para labs AWS)
- [Azure CLI](https://docs.microsoft.com/en-us/cli/azure/install-azure-cli) (para labs Azure)
- [VS Code](https://code.visualstudio.com/) (recomendado)
- Contas ativas na AWS e/ou Azure (Free Tier é suficiente para a maioria dos labs).
